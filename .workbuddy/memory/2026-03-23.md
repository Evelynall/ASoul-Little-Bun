## 添加鼠标按下图层回正速度调整项

**需求**：在图层管理 -> 鼠标按下图层 -> 图像调整中添加一个调整项，用于调整鼠标按下图层回正的速度。

**实现方案**：

1. **settings.py** - 添加默认设置项 `mouse_return_speed: 0.05`
   - 默认值0.05对应原来的硬编码衰减因子（1.0 - 0.05 = 0.95）

2. **input_handler.py** - 使用可配置的回正速度
   - 在 `MouseTracker.__init__()` 中从设置读取 `mouse_return_speed`
   - 在 `update_settings()` 中更新回正速度
   - 在 `update_mouse_position()` 中使用动态计算的 `return_factor`（1.0 - mouse_return_speed）

3. **custom_layer_dialog.py** - 添加UI控制项
   - 在鼠标按下图层的图像调整区域添加"回正速度"滑动条
   - 范围：1-200（对应0.001-0.2，单位：‰）
   - 在所有相关的信号连接、设置保存/加载、恢复默认等功能中集成新设置项

**效果**：
- 用户可以通过调整"回正速度"来控制鼠标按下图层跟随鼠标移动后的回正速度
- 数值越大，回正速度越快（更快回到中心位置）
- 数值越小，回正速度越慢（更平滑的回正效果）

## updater.py 添加配置文件合并策略

**问题**：自动更新程序原本直接覆盖所有文件，导致：
- `global_config.json` 完全跳过不更新 → 新增的全局配置键永远无法生效
- 角色 `config.json` 直接覆盖 → 用户的位置微调被新版默认值覆盖
- `custom_layers.json` 直接覆盖 → 用户自定义图层丢失

**解决方案**：在 `updater.py` 中实现三层合并策略：

1. **`deep_merge()`** — 递归字典合并，用于 `config.json` 和 `global_config.json`
   - 已有键保留用户值，只新增缺失的键
   - 嵌套字典递归合并

2. **`merge_custom_layers()`** — 按 `name` 字段去重的数组合并，用于 `custom_layers.json`
   - 保留用户已有的图层，只追加新版本中不存在的图层

3. **`merge_config_file()`** — 统一入口，根据文件名选择合并策略
   - 合并失败时自动回退到直接覆盖

**关键变更**：
- `global_config.json` 从跳过列表中移除，改为走合并流程
- `replace_files()` 方法中配置文件走合并路径，非配置文件仍直接复制
- 控制台输出区分 `"已更新"` 和 `"已合并更新"` 便于调试

## updater.py 实现自我更新（延迟替换）

**问题**：`updater.exe` 在跳过列表中排除了自身，永远无法被更新。

**核心挑战**：Windows 锁定正在运行的 exe，无法直接覆盖。

**解决方案**：延迟替换 + bat helper 脚本

1. **检测新版 updater**：遍历解压文件时，识别与当前 updater 同名的文件，暂存路径
2. **延迟替换**：所有其他文件替换完成后，创建 `_update_updater.bat` 辅助脚本
3. **bat 脚本逻辑**：
   - 通过 `tasklist` 轮询等待当前 updater 进程退出（PID 匹配）
   - updater 退出后 `copy /Y` 覆盖自身
   - 清理新版 updater 源文件和 `temp_update` 目录
   - 自删除 bat 脚本
4. **安全机制**：
   - `pending_self_update` 标记阻止 `cleanup()` 提前删除 temp 目录
   - 如果 bat 创建失败，回退到直接 `shutil.copy2`（PyInstaller 单文件 exe 有时可以覆盖自身）
